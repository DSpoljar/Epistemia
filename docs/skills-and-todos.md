## Current goals and changes:

- Change Project structure a bit:
When opening or creating a new project:

Have four tabs (or something akin to that) ready as follows:

1. Paper tab: Contains all the papers. Add Ability to upload papers (drag and drop) and also
add possibility to manually add them (title, authors, description, DOI / link to backtrace it).

2. Summaries of all papers. As mentioned before, the AI part should be easily added later. The user should be able to 
manually put his own notes and summaries (and having later the option to let the AI summarize for them!).

3. Claims or claim clusters, categorical per each paper added. 
The user should be able to categorize per ROW -> a claim/hypothesis (blue font), limitation (red font), methodology (yellow font), and implication (green font).
Later, the AI will and can help with that, but the user should also have the ability to do it on their own. Also, for each of these specific "rows", add the line and page 
of the paper to backtrace it.

4. Comparison tab: Make a table for each paper that lets one view all their core hypothesis, methodlogy, limitation, and implication. 

- Check the currently existing dummy project and adapt it to this structure as a first showcase. And as mentioned, see if it remains consistent with the project structure.
